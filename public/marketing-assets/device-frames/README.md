# Device frames

Drop transparent **PNG** mockup frames here (not SVG — PNG composites reliably
with Sharp). Each frame has a transparent "screen" window where the user's
screenshot is composited.

Expected files (names are referenced in `lib/device-frames.ts`):

| File | For | Notes |
|---|---|---|
| `macbook-pro-frame.png` | "Web app / dashboard" (laptop) | landscape, ~16:10 screen window |
| `iphone-14-frame.png` | "Mobile app" (phone) | portrait, ~19.5:9 screen window |
| `chrome-browser-frame.png` | "Landing page" (browser) | browser chrome at top, content area below |

The **floating** option needs no PNG — it's generated by Sharp (rounded corners +
soft shadow) and is also the automatic fallback when a frame file is missing, so
the feature works before you add these.

## After adding a PNG: set its screen rectangle
In `lib/device-frames.ts`, update that frame's `native` (the PNG's pixel size)
and `screen` ({x, y, w, h} of the transparent window, in the PNG's native px) so
the screenshot lands exactly inside the bezel. The defaults are placeholders.

Free sources: Mockuphone, Figma community mockups, or export from Figma/Sketch.
Prefer a clean transparent PNG with a clearly defined, axis-aligned screen area.
